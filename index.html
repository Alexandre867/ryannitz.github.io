Overview

Mersenne exponent set: (Me)

A Mersenne exponent is an integer n such that 2n-1 is prime. link

Mersenne number set (Mn)

A Mersenne number is an integer in the form 2n-1, where n is any positive integer. link

Mersenne prime number set (Mp)

A Mersenne prime is a prime number in the form 2p-1, where p is prime. link

Prime number set (Pn)

A prime number (or a prime) is a natural number greater than 1 that is not a product of two smaller natural numbers link

  • Mersenne exponent

    The set of Mersenne exponents is a subset of the prime number set defined in the above headers. When these specific exponents, n, are plugged into the Mersenne number with form 2n-1, the result will be a Mersenne prime.

  • Mersenne number

    Mersenne primes and Mersenne numbers are names after Marin Mersenne, a french scholar from the 1700s. Originally, Mersenne had the list of Mersenne Primes and Mersenne numbers incorrect. The set of primes contained composite numbers (Ex: M67, M257) while the set of composite numbers contained some prime numbers (Ex: M61, M281). Later, Edouard Lucas corrected for these smaller numbers using theories that we will show below.

    The mersenne number set is eclipsed by its own subset the Mersenne Primes. Much research is being conducted to find the next largest prime (GIMPS). Of course this would not be possible without Mersenne numbers.

    Mersenne numbers that are not prime, are composite Mersenne numbers. These are commonly factored to down to a product of primes. Ex: 211-1 = 2047 = 23x89

  • Mersenne prime number

    The star of the show, Mersenne numbers have a great spotlight in the prime number research space. A subset of the Mersenne numbers, these prime numbers are of the form 2p-1, where p is a prime.

//algorithm Sieve of Eratosthenes is
input: an integer n > 1.

let A be an array of Boolean values,

Sieve of Eratothenes

The Sieve of Eratosthenes is an ancient algorithm that eliminates all composite numbers up to a given number. It is thought that the Greek Mathmatician Eratosthenes of Cyrene is the one to have used it first.

Algorithm desc. The algorithm works by creating a list of booleans from 2 to the upper bound. It then does iterates to the upper bound, checking every element in the boolean array if it is true or false. If it hits an element that is true, then it will iterates over every multiple of that number setting the respective index to false in the boolean list. If it hits false, then another smaller integer was a multiple of that index and it was set to false, skipping to the next list element. At the end, iterate through the boolean list, save all of the indexes of the "true" values. These indexes are the primes from 2 to the upper bound.

Time complexity = O(n(log(logn)))

"Sift the Two's and Sift the Three's, The Sieve of Eratosthenes. When the multiples sublime, The numbers that remain are Prime." - Anonymous

To check if a Mersenne number is prime we can use the Lucas Lehmer primality test as defined below.

Pseudocode:

// Determine if Mp = 2p − 1 is prime for p > 2
Lucas–Lehmer(p)
var s = 4
var M = 2p − 1
repeat p − 2 times:
  s = ((s × s) − 2) mod M
if s == 0 return PRIME else return COMPOSITE

Implementation (Js):

//Logic for the Mersenne Prime test given the prime p
function lucasLehmerPrime(p){
  var mersenneVal = (2**p)-1;
  var nextVal = 4 % mersenneVal
  for(var i = 1; i < p-1; i++){
    nextVal = (nextVal*nextVal-2) % mersenneVal;
  }
  return nextVal==0;
}

Lucas–Lehmer primality test

LLPT was developed by Edouard Lucas in 1856, and later improved by Lucas in 1878, lastly by Derrick Lehmer in the 1930s. Its most famous application is in GIMPS, the Great Internet Mersenne Prime Search.

The LLPT algorithm has an efficient time-complexity for computational execution. Due to the fact that the iterations only execute up to p-2, the largest known prime would have taken 80 million execution cycles. This can be done relatively quickly on a modern computer. Where the algorithm is least efficient is in the mathmatical operation of 2p-1. This exponent calculation is sort of a hidden algorithm, it is executed in the background by iteratively multiplying 2 together. In applications like GIMP, other algorithms are implemented to square these huge numbers and divide related work loads.

Lucas-Lehmer sequence can be seen here (s2-2): 4, 42-2=14, 142-2=194, ...

Binary to Decimal Conversion

The actual implementation for the calculator above has been optimized to only calculate the difference from the previous number instead of a complete recalculation.

The binary representation of 2n-1 is n repitions of the digit 1. This can be extended to Mersenne primes where 2p-1 is p repitions of the digit 1, p being a Mersenne exponent.

This is because 2n creates a term with single digit of "1" followed by n digits of "0"
When 1 is taken away from this term, the term becomes n-1 digits of "1". In other words, the leading 1 is removed and all the "0"s become "1"s

Ex:
(100000-1)2 = (32-1)10
= 111112 = 3110

Getting back to 100000 from 11111 is more intuitive when we apply carry-overs from the adding operation. In base-2, the highest value is 1. Adding 1+1 would create a carry over.
Ex: 1+1 = 10

Likewise, 11111+1 = 100000, as the 1 carries all the way to the front while switching the trailing "1"s into "0"s.

Interesting facts.

  • Cryptography

    Large primes are used in encryption algorithms for cyber-security. The RSA encryption system is a PRIME example, using two large prime numbers to create a product that then requires prime-factorization. Note: Mersenne primes are not ideal for RSA encryption because they are guessed easily.

  • Javascript limiations

    I learned new limitations of javascript with big integer calculations. I had to research BigInt variable declaration and the javascript runtime memory sizes.

    BigInt allows us to calculate past 2^31, as 231 is a computers max integer. 231 seems odd but they use the first bit to represent the +/- of the integer. This allows computer to have an integer range of -231 to +231 for a total of 232 integer representations.

  • The Great Internet Mersenne Prime Search (GIMPS)

    GIMPS is an ongoing initiative to identify the largest known primes using Mersenne primes. GIMPS has a free program for download that requests assignments and work loads to be ran on the host system. If a computer finds a new prime that which is later confirmed through multiple checks and validations, the user and computer are documented and put in the record list.

    Many computer and hardware enthusiasts have probably heard of the GIMPS program "prime95", as it is heavily used for stress testing computers and processor voltage/power limits.

    The latest information can be found on their homepage: mersenne.org. One of the latest updates is from a contributer, they found a way to eliminate the need for a second primality test to be done when a new prime is found. This saves 95.5% of the workload that would have originally been required.
