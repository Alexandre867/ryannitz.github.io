The numbers created by 2n-1 are considered mersenne numbers while the prime numbers that are created using 2p-1 are considered mersenne primes. 2n-1 would create the value 2047. This is not a prime number so it would not be included in the mersenne primes set. To check if a mersenne number is prime we can use the Lucas Lehmer primality test.

Binary numbers with a prime number of digits will create a mersenne prime.
This is because 2n = "1" followed by n 0s
When 1 is taken away from this term, the number becomes "1" n times. So the leading largest 1 is removed and replaced with n-1 "1"
So 1000002-1 = 3210-1
= 111112 = 3110
Getting back to 100000 is more intuitive, as 11111+1 = 100000 as the 1 will carry over all the way to the front while switching the trailing "1"s into "0"s. (1+1 = 10 in binary)
